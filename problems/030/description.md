## 문제 설명

괄호 문자들로 이루어진 리스트 `괄호`를 받아 **짝이 올바르게 맞춰져 있는지** 판별해 말하기로 출력하세요. 괄호의 종류는 3가지: `(` `)`, `[` `]`, `{` `}`.

- 모든 괄호가 짝이 맞고 **중첩 순서까지** 올바르면 `올바름`
- 하나라도 어긋나면 `틀림`
- 빈 리스트는 올바른 것으로 간주 (`올바름`)

### 입력
- `괄호`: 괄호 문자 항목들의 리스트 (채점 시 자동 설정). 각 항목은 `"("`·`")"`·`"["`·`"]"`·`"{"`·`"}"` 중 하나

### 출력
- `올바름` 또는 `틀림`을 말하기

### 예시
입력: `["(", ")"]` → 출력: `올바름`
입력: `["(", "[", "]", ")"]` → 출력: `올바름`
입력: `[")", "("]` → 출력: `틀림`
입력: `["(", "]"]` → 출력: `틀림`

### 힌트
- 1차원 리스트를 **스택처럼** 써서 여는 괄호를 쌓고 닫는 괄호를 만날 때 맨 뒤 항목과 비교
- `자료`의 `항목 추가하기`·`N번째 항목`·`N번째 항목 삭제하기`·`항목 수` 블록
- 보조 스택 리스트, 순회 위치·현재값·오류 플래그용 변수는 직접 만들어 쓰세요

<details>
<summary>자세한 접근법</summary>

- 빈 스택 리스트를 준비. 오류 플래그를 `0`으로 초기화
- 리스트를 앞에서부터 순회:
  - 여는 괄호(`(`·`[`·`{`)면 스택의 **뒤에 추가**
  - 닫는 괄호면:
    - 스택이 비어 있으면 오류 플래그 = 1
    - 아니면 스택 **마지막 항목**이 짝(`(`↔`)`, `[`↔`]`, `{`↔`}`)인지 확인
      - 맞으면 스택 마지막 항목 삭제
      - 안 맞으면 오류 플래그 = 1
- 순회 끝난 뒤:
  - 오류 플래그 = 1 **또는** 스택이 비어있지 않음 → `틀림`
  - 둘 다 아니면 → `올바름`
- 오류 플래그가 `1`이 된 뒤엔 나머지 반복을 건너뛰도록 `...인 동안 반복`의 조건에 포함 가능

</details>
