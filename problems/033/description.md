## 문제 설명

**요세푸스(Josephus) 문제**: `N`명이 원형으로 앉아 번호 `1..N`을 가지고 있습니다. 1번부터 시작해 **`K`번째** 사람을 제거하고, 제거된 자리의 다음 사람부터 다시 K번째 사람을 제거하는 과정을 반복합니다. 마지막에 남는 한 사람의 **번호**를 말하기로 출력하세요.

예시 (N=5, K=2):
- 1번부터 2번째 → 2번 제거. 남은 `[1,3,4,5]`
- 3번부터 2번째 → 4번 제거. 남은 `[1,3,5]`
- 5번부터 2번째 → 1번 제거. 남은 `[3,5]`
- 3번부터 2번째 → 5번 제거. 남은 `[3]` → **생존자 3번**

### 입력
- `N`: 1 이상의 자연수 (사람 수, 채점 시 자동 설정)
- `K`: 1 이상의 자연수 (제거 간격, 채점 시 자동 설정)

### 출력
- 마지막 생존자의 번호를 말하기

### 예시
입력: `N = 5`, `K = 2` → 출력: `3`
입력: `N = 7`, `K = 3` → 출력: `4`
입력: `N = 1`, `K = 1` → 출력: `1` (한 명뿐이라 그대로 생존)

### 힌트
- 변수 `N`, `K`, `시작`, `i`는 **기본으로 제공**됩니다
- 리스트 `남은사람들`도 **기본으로 제공**됩니다 (처음엔 비어 있음)
- 핵심 아이디어: 원형 배열을 **배열 + 나머지 연산**으로 시뮬레이션
  - **1단계 (초기화)**: `i = 1..N` 반복하며 `남은사람들`에 `i`를 추가 → `[1, 2, ..., N]`
  - **2단계 (제거 반복)**: `시작 = 0`으로 두고, `남은사람들의 항목 수 > 1` 인 동안 반복:
    - `시작 = (시작 + K - 1)을 남은사람들의 항목 수로 나눈 나머지` (0-based 인덱스 갱신)
    - `남은사람들의 (시작 + 1)번째 항목 삭제` (1-based 위치로 변환해 삭제)
- 반복이 끝나면 `남은사람들의 1번째 항목`을 말하기 (마지막 생존자)
- **왜 `-1`을 더하나?** K번째 사람을 세려면 `K-1`칸 이동해야 합니다 (현 자리부터 포함해 K번째).
- **왜 항목 수로 나머지?** 원형이므로 끝을 넘으면 처음으로 돌아와야 합니다.
- `흐름` 카테고리의 `N번 반복하기`·`...인 동안 반복하기`, `자료`의 `항목 추가하기`·`N번째 항목 삭제하기`·`항목 수`·`N번째 항목`, `계산`의 `나머지`·`더하기`·`빼기` 블록을 활용하세요
