## 문제 설명

**행복수(Happy Number)**: 자연수 `N`의 각 자릿수를 제곱해 더하는 과정을 반복하면 결국 `1`에 도달하는 수를 **행복수**라고 부릅니다. 반대로 과정이 이미 지나온 값으로 돌아와 무한 순환에 빠지면 **불행수**입니다.

예시 (19는 행복수):
- 19 → 1² + 9² = 82
- 82 → 8² + 2² = 68
- 68 → 6² + 8² = 100
- 100 → 1² + 0² + 0² = **1** ✓

예시 (4는 불행수 — 자기 자신으로 돌아옴):
- 4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4 → …

`N`이 행복수면 `행복수`, 아니면 `불행수`를 말하기로 출력하세요.

### 입력
- `N`: 1 이상의 자연수 (채점 시 자동 설정)

### 출력
- `행복수` 또는 `불행수`를 말하기

### 예시
입력: `N = 1` → 출력: `행복수`
입력: `N = 19` → 출력: `행복수`
입력: `N = 4` → 출력: `불행수`

### 힌트
- 지금까지 거쳐온 값을 기록할 보조 리스트가 필요 (직접 만들어 쓰세요)
- 자릿수 제곱합용 보조 변수들도 직접 만드세요
- `계산`의 `나누기의 몫`·`나누기의 나머지`, `자료`의 `항목 추가하기` 블록

<summary>자세한 접근법</summary>

- 바깥 반복: 아직 결론(행복/불행)이 나지 않은 동안
  - `N = 1` → 결과는 행복수, 탈출
  - 아니면 `N`이 이력 리스트에 이미 있는지 검사
    - 있으면 순환 감지 → 결과는 불행수, 탈출
    - 없으면 이력에 `N` 추가 후 자릿수 제곱합으로 `N` 갱신
- 안쪽 반복 (자릿수 제곱합):
  - 합 변수를 `0`, 남은 = `N`
  - 남은 > 0 인 동안: 자리 = 남은 `mod` 10, 합 += 자리×자리, 남은 = 남은 `div` 10
  - 반복 후 `N`에 합을 대입
- **주의**: 결과 문자열 변수를 쓸 경우 반드시 `""`로 초기화 (엔트리 기본값 `0`이면 비교 엉킴)
