## 문제 설명

리스트 `숫자들`에서 **엄격히 증가하는 부분수열(LIS: Longest Increasing Subsequence)**의 **길이**를 구해 말하기로 출력하세요.

- "부분수열"은 연속되지 않아도 되며 원래 순서는 지킵니다 (일부 항목을 건너뛰기 가능)
- "엄격히 증가"는 앞 항목 < 뒤 항목 (같거나 작으면 안 됨)

### 입력
- `숫자들`: 1개 이상의 정수 리스트 (채점 시 자동 설정)

### 출력
- LIS의 길이를 말하기

### 예시
입력: `[3,1,4,1,5,9,2,6]` → 출력: `4` (예: `[1,4,5,9]` 또는 `[1,2,6]`은 `[1,4,5,6]`의 대안)
입력: `[1,2,3,4,5]` → 출력: `5` (전체가 LIS)
입력: `[5,4,3,2,1]` → 출력: `1` (모두 내림차순이라 단일 항목이 최장)

### 힌트
- 리스트 `숫자들`은 **기본으로 제공**됩니다 (채점 시 값이 교체됨)
- 리스트 `길이`도 **기본으로 제공**됩니다 (DP 보조용)
- 변수 `i`, `j`, `최댓값`은 **기본으로 제공**됩니다
- 핵심 아이디어: 각 위치 `i`마다 "`i`에서 **끝나는** LIS의 길이"를 `길이의 i번째 항목`에 저장하는 O(N²) DP
  - **1단계 (초기화)**: `길이`를 비우고 `숫자들의 항목 수`만큼 `1`을 추가 (자기 자신 하나만 고르면 길이 1)
  - **2단계 (DP 채우기)**: `i = 2..숫자들의 항목 수` 바깥 반복
    - 안쪽 반복 `j = 1..i-1`:
      - 만약 `숫자들의 j번째 항목 < 숫자들의 i번째 항목` **그리고** `길이의 j번째 항목 + 1 > 길이의 i번째 항목` 이면:
        - `길이의 i번째 항목`을 `길이의 j번째 항목 + 1` 로 갱신 (더 긴 수열을 이어 붙일 수 있음)
  - **3단계 (최댓값)**: `길이` 리스트 전체를 훑어서 최댓값을 `최댓값` 변수에 저장
- 마지막에 `최댓값` 말하기
- `흐름` 카테고리의 `N번 반복하기`를 **이중 중첩**, `자료`의 `항목 추가하기`·`N번째 항목을 X로 바꾸기`·`N번째 항목`, `판단`의 `그리고`·`<`·`>` 블록을 활용하세요
