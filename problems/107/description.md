## 문제 설명

동전 단위가 들어있는 리스트 `동전들`과 목표 `금액`이 주어집니다. `금액`을 **정확히** 만들기 위해 필요한 **최소 동전 개수**를 구해 말하기로 출력하세요.

- 동전은 같은 단위를 여러 번 사용 가능
- 언제나 `금액`을 만들 수 있도록 `1`원짜리가 포함됨 (또는 만들 수 있는 조합이 존재함을 보장)
- 그리디로는 해결되지 않을 수 있음 (예: 동전 `[1, 3, 4]`로 금액 `6` → 그리디 `3개`, 최적 `2개`)

### 입력
- `동전들`: 1 이상의 서로 다른 정수 리스트 (채점 시 자동 설정)
- `금액`: 0 이상의 정수 (채점 시 자동 설정)

### 출력
- `금액`을 만드는 최소 동전 개수를 말하기

### 예시
`동전들 = [1, 2, 5], 금액 = 11` → 출력: `3` (5+5+1)
`동전들 = [1, 3, 4], 금액 = 6` → 출력: `2` (3+3)
`동전들 = [1], 금액 = 7` → 출력: `7`
`동전들 = [2, 5], 금액 = 0` → 출력: `0`

### 힌트
- DP 배열로 `0..금액` 각 값을 만드는 최소 동전 개수를 저장
- 점화식: `dp[i] = min(dp[i - 동전]) + 1` (모든 동전 단위에 대해)
- 초기값: `dp[0] = 0`, 나머지는 큰 값으로 설정
- DP 배열은 직접 만들어 쓰세요
