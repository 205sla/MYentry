# Entry Algorithm Platform

Entry(엔트리) 블록 코딩 기반 알고리즘 문제 풀이 플랫폼입니다. 각 문제의 지문을 좌측 패널에서 보고, 우측의 엔트리 에디터로 블록을 조립하여 풀이합니다. 제출 시 로컬에서 테스트 케이스를 자동 채점합니다.

## 실행 방법

```bash
npm install
npm start
```

브라우저에서 `http://localhost:3000` 접속 (또는 `start.bat` 더블클릭)

> 반드시 HTTP 서버를 통해 접속해야 합니다. `index.html`을 직접 더블클릭하면 브라우저 보안 정책 때문에 동작하지 않습니다.

## 디렉터리 구조

```
MYentry/
├── server.js                 # Express 서버 (포트 3000)
├── problems/                 # 문제 데이터 (번호별 디렉터리)
│   ├── 001/
│   │   ├── meta.json         # {"title": "..."}       (필수)
│   │   ├── description.md    # Markdown 문제 설명      (선택)
│   │   ├── project.ent       # 기본 프로젝트 파일      (선택)
│   │   └── tests.json        # 테스트/제출 케이스      (선택)
│   ├── 002/
│   └── ...
└── public/
    ├── index.html            # 메인 화면 (문제 선택)
    ├── editor.html           # Entry 블록 코딩 에디터
    ├── media/                # 커서 파일
    ├── sprites/              # 로컬 번들 스프라이트 (10종) + catalog.json
    └── lib/                  # Entry 전용 라이브러리 (로컬 번들)
        ├── entry-js/         # 엔트리 코어 엔진
        ├── entry-tool/       # 팝업 UI
        ├── entry-paint/      # 그림 편집기
        ├── entry-lms/        # 모달 시스템
        ├── sound-editor/     # 소리 편집기
        └── legacy-video/     # 비디오 모듈
```

## 화면 구성

### 메인 화면 (`/`)
- `problems/` 디렉터리를 스캔하여 `meta.json`이 있는 번호 디렉터리를 문제 카드로 표시
- 문제 클릭 시 에디터로 이동 (`/editor.html?problem=N`)
- "자유 모드로 시작" 버튼으로 문제 없이 에디터 열기 가능

### 에디터 화면 (`/editor.html?problem=N`)
- **좌측 문제 패널**: Markdown 렌더링된 문제 설명. 스크롤 가능, 스플리터 드래그로 40~800px 범위에서 리사이즈 가능
- **우측 엔트리 워크스페이스**: 블록 카테고리, 스테이지, 블록 조립 보드
- **헤더 버튼**:
  - `블록` / `파이썬` — 블록 모드와 파이썬 텍스트 코딩 모드 전환
  - `🔍 테스트하기` (파란색) — `tests.json`의 `test` 케이스(공개)로 채점
  - `✓ 제출하기` (주황색) — `tests.json`의 `submit` 케이스(숨김)로 채점
- **Entry 내장 버튼**: 스테이지 아래 "▶ 시작하기"/"■ 정지하기"로 실행/중지

### 비활성화된 블록 카테고리
알고리즘 플랫폼에 적합하도록 다음 카테고리는 숨겨져 있습니다:
- 데이터 분석, 인공지능, 확장, 하드웨어

## 문제 추가 방법

`problems/NNN/` 형식의 디렉터리를 만들고 아래 파일들을 배치합니다 (번호는 3자리 0패딩: `001`, `042`, `100`).

**서버 재시작 불필요** — 파일을 추가/수정하고 브라우저를 새로고침하면 즉시 반영됩니다.

### `meta.json` (필수)
```json
{ "title": "문제 제목" }
```

선택 필드 `sprites` — 오브젝트/모양 추가 팝업에 노출할 스프라이트 ID 목록 (배열 순서 유지). 미지정 시 전체 10종 노출.

```json
{
  "title": "문제 제목",
  "sprites": ["sp01", "sp03", "sp07"]
}
```

| `sprites` 값 | 팝업 동작 |
|--------------|----------|
| 필드 없음 | 전체 10종 (기본) |
| `[]` | 아무것도 안 보임 (명시적 차단) |
| `["sp01","sp03"]` | 지정된 ID만, 배열 순서대로 |
| 존재하지 않는 ID 포함 | 무효 ID 자동 스킵 |

사용 가능한 ID: `sp01`~`sp10` — 자세한 내용은 `public/sprites/catalog.json` 참조.

### `description.md` (선택)
Markdown 형식의 문제 설명. 지원 문법:
- 제목: `#`, `##`, `###`
- 강조: `**굵게**`, `*기울임*`
- 인라인 코드: `` `코드` ``
- 목록: `- 항목`, `1. 항목`
- 코드 블록: ```` ```언어 ... ``` ````

### `project.ent` (선택)
엔트리 공식 사이트([playentry.org](https://playentry.org))에서 "내 컴퓨터에 저장하기"로 내보낸 기본 프로젝트 파일. 없으면 빈 프로젝트로 시작합니다.

### `tests.json` (선택)
테스트 케이스. `test` 배열은 "테스트하기" 버튼용(공개), `submit` 배열은 "제출하기" 버튼용(숨김). 파일이 없으면 두 버튼이 모두 표시되지 않습니다.

```json
{
  "test": [
    {
      "name": "기본 케이스",
      "setup": {
        "variables": { "변수명": 값 },
        "lists": { "리스트명": [값, 값, ...] }
      },
      "expected": {
        "say": ["말하기로 출력할 내용"],
        "variables": { "변수명": "기대값" },
        "lists": { "리스트명": [기대, 값, ...] }
      },
      "timeout": 5000
    }
  ],
  "submit": [ /* 동일 형식, 숨겨진 케이스 */ ]
}
```

#### `setup` — 실행 전 초기값
| 필드 | 설명 |
|------|------|
| `variables` | 변수 초기값 (변수명 → 값) |
| `lists` | 리스트 초기 내용 (리스트명 → 배열) |

#### `expected` — 실행 후 기대값
| 필드 | 검증 내용 |
|------|---------|
| `say` | 말하기/묻기/외치기/생각하기 블록으로 출력한 메시지 배열 (부분 일치) |
| `variables` | 변수 최종 값 (엔진 정지 직전 기준) |
| `lists` | 리스트 최종 상태 (엔진 정지 직전 기준) |

#### 채점 동작
1. 버튼 클릭 시 서버에서 해당 모드의 케이스들을 가져옴
2. 각 케이스마다:
   - `setup` 적용 → 엔진 실행
   - 모든 블록 실행 종료 또는 5초 타임아웃 대기
   - **0.3초 대기** (상태 안정화)
   - 변수/리스트 최종 상태 캡처 (엔진 정지 시 스냅샷으로 복원되므로 정지 직전에 캡처)
   - 엔진 정지 후 `expected`와 비교
3. 실행 중 경고(예: 빨간색 블록)가 발생하면 해당 케이스는 **오류**로 표시하고 건너뜀
4. 결과 모달 표시:
   - **테스트하기**: 실제 케이스 이름과 실패 시 기대값/실제값 차이 표시
   - **제출하기**: 케이스 이름을 `테스트 1`, `테스트 2`... 로 비공개, 실패 상세 비공개 (오류는 공개)

## API

서버는 다음 엔드포인트를 제공합니다. 모든 응답은 JSON.

| 엔드포인트 | 설명 |
|-----------|------|
| `GET /api/problems` | 전체 문제 목록 `[{id, title}]` |
| `GET /api/problems/:id/meta` | 문제 제목 + 설명 `{id, title, description}` |
| `GET /api/problems/:id/tests?mode=test\|submit` | 테스트 케이스 `{mode, cases}` |
| `GET /api/problems/:id/has-tests` | 테스트 존재 여부 `{hasTests: boolean}` |
| `GET /api/problems/:id` | 프로젝트 데이터 (`project.ent`에서 추출) |

## 기술 스택

- **Frontend**: EntryJS (엔트리 블록 코딩 엔진)
- **Backend**: Node.js + Express
- **Entry 라이브러리**: entry-js, entry-tool, entry-paint, entry-lms, sound-editor, legacy-video

## 특징

- **오프라인 대응**: Entry 관련 라이브러리는 모두 `public/lib/`에 로컬 번들링. 범용 라이브러리(jQuery, React 등)만 CDN 사용.
- **로컬 채점**: 모든 채점은 클라이언트에서 실행 → 서버 부하 없음.
- **상태 관리 없음**: 사용자 계정, 저장, 제출 이력 등 일체 없음. 순수하게 브라우저 세션 내에서만 동작.
- **핫 리로드 가능**: 문제 파일 수정 시 서버 재시작 없이 즉시 반영.
